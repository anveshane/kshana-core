# Tracking Progress on Multi-Item Tasks

When you have multiple items to create—several characters, many scenes, a series of images—tracking progress helps you stay organized and helps users understand where things stand.

## Why Track Progress

**Resume If Interrupted**
Projects may span multiple sessions. Know what's done and what remains.

**Show Users Where You Are**
Users appreciate visibility into progress. "4 of 8 scenes complete" is informative.

**Ensure Nothing Is Missed**
With many items, it's easy to lose track. A checklist prevents gaps.

**Manage Dependencies**
Some items depend on others. Tracking helps sequence work correctly.

## What to Track

### Items to Create

List everything that needs to be done:
- All characters that need profiles
- All settings to describe
- All scenes to develop
- All images to generate
- All video clips to produce

### Status of Each Item

Mark where each item stands:
- **Pending**: Not yet started
- **In Progress**: Currently working on it
- **Needs Review**: Done, awaiting approval
- **Approved**: User approved, can proceed
- **Completed**: Fully done

### Dependencies

Note what blocks what:
- Character references before scene images
- Scene descriptions before scene images
- All scenes before video compilation

## How to Track

Think of it like maintaining a checklist. At any point, you should be able to answer:
- What's done?
- What's in progress?
- What's next?
- What's blocked?

### Starting a Phase

When beginning work that involves multiple items:
1. List all items to create
2. Note any dependencies
3. Mark everything as pending
4. Select the first item to work on

### During Work

As you work:
1. Mark current item as in progress
2. Complete the work
3. If it needs approval, mark as "needs review"
4. Once approved, mark as completed
5. Move to next item

### Handling Interruptions

If work is interrupted:
1. Current state is preserved
2. On resumption, check what's done
3. Continue from where you left off

## Presenting Progress to Users

Keep users informed without overwhelming them.

### Summary Form
"I've completed 4 of 8 character profiles. Currently working on Elena. Marco and the background characters remain."

### When Seeking Approval
"Here are the scene descriptions for your review:
- Scene 1: The Discovery (ready)
- Scene 2: The Confrontation (ready)
- Scene 3: The Escape (ready)
...
Would you like to review these before I continue?"

### After Completion
"All 6 scenes have been described and approved. Ready to proceed to image generation."

## Common Patterns

### Batch Creation
Creating multiple similar items (5 characters, 8 scenes):
1. List all items at start
2. Create each one
3. Present batch for approval
4. Track which approved, which need revision
5. Revise rejected items
6. Continue once all approved

### Sequential Dependencies
Items that must be done in order:
1. Story → Characters → Settings → Scenes
2. Complete each phase before next
3. Track both item progress and phase progress

### Parallel Work
Items that can be done in any order:
1. Multiple character profiles
2. No dependencies between them
3. Track individually but can work flexibly

### Revision Cycles
When items need multiple rounds:
1. Track original creation
2. Track revision requests
3. Track revision completion
4. May go through multiple cycles

## Tips

**Start with the List**
Before creating anything, enumerate what you'll create. This catches missing items early.

**Update Frequently**
Track as you go, not after. Keeps state accurate.

**Be Specific**
"Scene 3" not "a scene." Clear identification prevents confusion.

**Show Progress Naturally**
Work progress updates into your responses. "Moving on to the third character, Maya..."

**Handle Revisions Gracefully**
Approved items may need changes later. Update tracking accordingly.
