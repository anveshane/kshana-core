# Understanding Project State

Before working on a project, understand what exists. Projects may be new, partially complete, or returning from interruption. Context discovery helps you pick up appropriately.

## Why Discovery Matters

**Avoid Duplication**
Don't recreate what already exists. Check first.

**Maintain Consistency**
Existing content establishes patterns. New content should match.

**Resume Correctly**
Projects may have been interrupted. Pick up where things left off.

**Understand User Intent**
What exists reveals what the user is building.

## What to Discover

### Project Structure

What does this project contain?
- Story or content
- Characters defined
- Settings established
- Scenes planned
- Images generated
- Videos created

### Current State

Where is the project in its workflow?
- Just starting (nothing exists)
- Mid-creation (some elements done)
- Awaiting approval (content ready for review)
- Ready for generation (approved, need images/video)
- Complete (all elements finished)

### Quality and Consistency

What patterns are established?
- Writing style used
- Visual style established
- Level of detail in descriptions
- Naming conventions

### Pending Work

What still needs to be done?
- Items marked as incomplete
- Phases not yet reached
- Revisions requested but not made

## How to Discover

### Check Project State

Look at what's stored in the project:
- Read project configuration
- Check content files
- Review any saved progress

### Analyze Existing Content

If content exists, understand it:
- What's the story about?
- Who are the characters?
- What visual style is used?
- What's been approved vs. pending?

### Identify Gaps

Compare what exists to what's needed:
- Missing characters mentioned but not defined
- Settings referenced but not described
- Scenes listed but not developed
- Descriptions without images

### Note Dependencies

Understand what blocks what:
- Can't generate images without descriptions
- Can't generate video without images
- Can't proceed without approval

## Discovery Questions

Ask yourself these questions at project start:

**Is this a new project?**
- If yes: Start from the beginning
- If no: Understand existing state

**What exists?**
- Story? Characters? Settings? Scenes?
- Images? Videos?
- What quality and completeness?

**What's the user's goal?**
- Creating from scratch?
- Continuing previous work?
- Modifying existing content?
- Adding to complete project?

**What's next?**
- What's the logical next step?
- What does the user expect to happen?

## Common Situations

### New Project
Nothing exists. Start with understanding user input and building from scratch.

### Returning to Work
Content partially complete. Find where things stopped, continue from there.

### Modification Request
Project complete or partial, user wants changes. Understand what exists, apply changes, maintain consistency.

### Expansion
Project has some content, user wants more. Build on existing foundation.

## Presenting Your Understanding

After discovery, confirm your understanding with the user:

### For New Projects
"I understand you want to create a documentary about climate change. I'll start by researching the topic and developing an outline. Does this match your expectations?"

### For Returning Projects
"I see you have a story with 4 characters defined and 6 scenes planned. Scenes 1-3 have images, scenes 4-6 are still descriptions only. Would you like me to continue generating images for the remaining scenes?"

### For Modifications
"You have a complete short video about a detective. You'd like to make the ending more ambiguous. I'll revise scenes 7 and 8 to adjust the resolution. Is that correct?"

## Tips

**Discover Before Acting**
Don't assume—check. Even small projects may have relevant state.

**Be Thorough but Efficient**
Check what matters, skip what doesn't. Not every detail needs examination.

**Surface Confusion**
If state is unclear or inconsistent, ask the user rather than guessing.

**Update as You Work**
Discovery isn't just at the start. As you work, you learn more about the project.

**Respect Existing Work**
What exists represents user effort. Don't casually discard or overwrite.
