# Product & Features Phase

This phase details the products and their key features.

## Phase Goal

Document products and features with visual detail for generation.

## Current State

{{PHASE_STATE}}

## Input Context

Use `list_project_files` to see available content, then `read_file` to access what you need.

The value proposition should be available at `plans/value_proposition.md`. Read it to develop product and feature details.

## Artifacts in This Phase

- **Products**: Detailed product descriptions
- **Features**: Key feature breakdowns

## Product Documentation

For each product, detail:

### Basic Information
- Product name
- Category/type
- Price point (if relevant)
- Variants (colors, sizes)

### Physical Description
- Dimensions and size
- Materials and textures
- Colors and finishes
- Weight and feel

### Visual Style
- Design aesthetic
- Key visual elements
- Brand colors
- Logo placement

### Key Features List
- Top 3-5 features
- What makes each special
- How each works

## Feature Documentation

For each feature, detail:

### Feature Description
- What it does
- How it works
- Technical details (if relevant)

### Benefit Translation
- What problem it solves
- How it helps the user
- Emotional benefit

### Visual Representation
- How to show it working
- Before/after if applicable
- Best camera angle

### Demonstrability
- Can this be shown in action?
- What's the demo scenario?
- What motion demonstrates it?

## Workflow

For products:
1. Document physical appearance
2. List key features
3. Define visual style
4. Present for approval

For features:
1. Describe functionality
2. Translate to benefits
3. Plan visual representation
4. Present for approval

## Quality Criteria

Before completing this phase:
- [ ] Products fully described
- [ ] Features clearly defined
- [ ] Visual details sufficient for generation
- [ ] Benefits are articulated
- [ ] Each item individually approved
