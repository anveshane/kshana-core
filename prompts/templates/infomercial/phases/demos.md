## Demo Sequence Components

For each demo, define:

### Demo Overview
- Which product(s) featured
- Which feature(s) demonstrated
- Duration estimate

### Setup
- Starting situation
- What viewer sees first
- Any "before" state

### Action
- What happens in the demo
- Step by step if complex
- Key moment to capture

### Result
- What's the outcome
- Visible benefit
- "After" state if applicable

### Visual Direction
- Camera angle
- Lighting needs
- Environment/setting

## Demo Types

### Usage Demo
- Show product in normal use
- Natural interaction
- Ease of use visible

### Feature Demo
- Focus on specific feature
- Show it working
- Highlight capability

### Comparison Demo
- Before/after
- With/without
- Us vs competition (careful)

### Result Demo
- Show end result
- Proof of effectiveness
- Transformation visible

## Demo Prioritization

1. Most compelling first
2. Most important feature
3. Most believable claim
4. Best visual impact

## Workflow

1. Review script for demo needs
2. Identify demo opportunities
3. Plan each demo sequence
4. Ensure features covered
5. Present for approval

## Quality Criteria

Before completing this phase:
- [ ] All key features have demos
- [ ] Demo actions are clear
- [ ] Visual direction is specific
- [ ] Sequence makes sense
- [ ] Each demo individually approved
