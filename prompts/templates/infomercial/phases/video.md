# Video Generation Phase

This phase generates demo video clips.

## Phase Goal

Create compelling video demonstrations from demo images.

## Current State

{{PHASE_STATE}}

## Input Context

Use `list_project_files` to see available content, then `read_file` to access what you need.

Demo sequences are in the `demos/` directory. Demo images are tracked in `project.json` with their artifact IDs.

## Artifacts in This Phase

- **Demo Videos**: Animated demonstration clips

## Demo Video Requirements

### Show the Demo
- Feature in action
- Clear demonstration
- Visible result/benefit

### Timing
- Match script timing
- Enough time to see demo
- Maintains overall pace

### Quality
- Commercial-ready
- Smooth, professional motion
- No artifacts

## Motion Types for Product Demos

### Product Action
- Product feature activating
- Using the product
- Feature in motion

### Human Interaction
- Hands using product
- Person enjoying benefit
- Natural-looking action

### Result/Transform
- Before to after
- Change happening
- Benefit appearing

### Detail Focus
- Zoom to feature
- Highlight capability
- Draw attention

## Workflow

For each demo image:
1. Review demo sequence details
2. Plan motion and timing
3. Generate video clip
4. Present for approval
5. Allow regeneration

## Quality Guidelines

### Motion Quality
- Smooth and professional
- No jerky movements
- Commercial standard

### Product Consistency
- Same product throughout
- Matches reference shots
- Recognizable

### Demo Clarity
- Action is clear
- Feature is shown working
- Benefit is visible

## User Approval

This is an EXPENSIVE phase:
- Show motion plan
- Confirm before generation
- Play clip for review
- Allow refinement

## Quality Criteria

Before completing this phase:
- [ ] All demo videos generated
- [ ] Demonstrations are clear
- [ ] Product is consistent
- [ ] Motion is professional
- [ ] Each clip individually approved
