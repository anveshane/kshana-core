# Story Breakdown Phase

This phase breaks down the complete story into its component parts: characters, settings, and scenes.

## Phase Goal

Extract and detail all characters, settings, and scenes from the story to prepare for visual generation.

## Current State

{{PHASE_STATE}}

## Input Context

Use `list_project_files` to see available content, then `read_file` to access what you need.

The story should be available at `plans/story.md`. Read it to extract characters, settings, and scenes.

## Artifacts in This Phase

- **Characters**: Individual character descriptions with visual details
- **Settings**: Location descriptions with visual details
- **Scenes**: Scene-by-scene breakdown with visual direction

## Workflow

### 1. Character Extraction
For each character in the story:
- Create a detailed physical description
- Note distinctive features, clothing, accessories
- Define their visual style and color palette
- Include personality traits that affect appearance

### 2. Setting Extraction
For each location in the story:
- Create a detailed environmental description
- Note key visual elements and props
- Define lighting and atmosphere
- Include time-of-day variations if relevant

### 3. Scene Breakdown
For each scene in the story:
- Identify which characters appear
- Identify which setting is used
- Describe the key visual moment
- Note camera angle suggestions
- Include emotional tone and lighting

## Per-Item Approval

Each character, setting, and scene requires individual approval before proceeding:
- Present one item at a time
- Allow for refinement based on feedback
- Track approval status for each item

## Visual Consistency Guidelines

### Characters
- Use consistent terminology for features
- Note any changes in appearance throughout story
- Include reference poses useful for generation

### Settings
- Maintain consistent visual vocabulary
- Note connecting elements between settings
- Consider how settings evolve during the story

### Scenes
- Reference characters by their exact names
- Reference settings by their exact names
- Be specific about positioning and actions

## Quality Criteria

Before completing this phase:
- [ ] All characters have detailed visual descriptions
- [ ] All settings have detailed visual descriptions
- [ ] All scenes reference specific characters and settings
- [ ] Each item has been individually approved
- [ ] Visual consistency is maintained across descriptions
